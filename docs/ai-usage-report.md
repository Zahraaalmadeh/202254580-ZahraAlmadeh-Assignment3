# AI Usage Report

## 1. Introduction
This report explains how AI tools were used during the development and refinement of the portfolio website project. The purpose of this report is to document the role of AI clearly, honestly, and responsibly.

AI was used as a support tool to improve code quality, identify weaknesses, strengthen documentation, and help refine the final submission. However, the final decisions, review, editing, and validation remained the responsibility of the student.

## 2. Purpose of Using AI
AI was used to support the development process in the following ways:
- Improve clarity and structure of code
- Identify possible bugs or weak areas
- Enhance technical documentation
- Strengthen the README file
- Produce a more detailed and reflective AI usage report
- Suggest usability and validation improvements

The main goal was not to let AI complete the work automatically, but to use it as an assistant for feedback, refinement, explanation, and idea generation.

## 3. Areas Where AI Was Used

### 3.1 Code Review and Improvement
AI was used to inspect the HTML, CSS, and JavaScript files and identify areas that could be improved. This included:
- Reviewing code organization
- Improving readability
- Making validation logic clearer
- Suggesting more maintainable function structures
- Highlighting possible issues such as incorrect asset paths

### 3.2 Debugging Support
AI was specifically helpful in reviewing the contact form validation logic, especially after previous grading feedback indicated that email validation did not work as intended.

AI assistance was used to:
- Re-check the email validation regular expression
- Improve validation flow
- Improve user-facing validation messages
- Reduce the chance of repeating the same mistake in the next assignment

### 3.3 Documentation Writing
AI was used to help generate and improve project documentation, including:
- README content
- Technical documentation
- AI usage report structure and wording

This was useful for making the documentation more complete, more professional, and more aligned with academic expectations.

### 3.4 UX and Feature Suggestions
AI also provided suggestions for improving user experience, such as:
- clearer status messages
- better explanation of setup steps
- stronger guidance for the user
- more detailed discussion of limitations and future improvements

## 4. Specific AI Contributions
The following are examples of the kind of assistance AI provided:

- Suggested a more reliable email validation function
- Recommended expanding setup instructions in the README
- Helped explain technical features in a more structured way
- Helped identify missing details in technical documentation
- Suggested including risks and mitigation strategies in the AI report
- Provided better wording for academic and technical descriptions

## 5. Human Role in the Process
Although AI was used, the project still required substantial human involvement.

The student was responsible for:
- Choosing the website structure
- Writing and organizing the portfolio content
- Selecting the projects to display
- Understanding how the code works
- Deciding which AI suggestions were useful
- Rejecting or modifying suggestions when necessary
- Reviewing the final output for correctness and assignment alignment

This means AI supported the process, but it did not replace learning, coding decisions, or final responsibility.

## 6. Benefits of Using AI

### 6.1 Increased Efficiency
AI helped speed up certain tasks, especially:
- rewriting documentation
- checking for common front-end issues
- refining validation logic
- improving wording and explanations

### 6.2 Better Documentation Quality
One of the strongest benefits was documentation support. AI helped transform basic documentation into more complete and professional markdown files.

### 6.3 Improved Error Detection
AI helped identify weaknesses that may have been overlooked, such as:
- weak email validation
- missing setup instructions
- insufficient technical explanation
- limited AI reflection detail

### 6.4 Better Reflection and Academic Presentation
AI also helped improve the depth of reflection in the documentation by encouraging discussion of:
- risks of AI usage
- limitations of generated output
- the importance of manual review

## 7. Risks of AI Usage
Using AI in software development also introduces several risks. These must be acknowledged clearly.

### 7.1 Incorrect or Incomplete Code
AI-generated code may look correct but still contain:
- logic errors
- edge-case failures
- weak validation
- poor assumptions
- compatibility problems

This is particularly important in academic settings because accepting AI output without testing can lead to avoidable mistakes.

### 7.2 Over-Reliance on AI
A major risk is depending too heavily on AI without fully understanding the output. This can reduce:
- critical thinking
- debugging ability
- independent problem solving
- long-term learning

### 7.3 Generic Documentation
AI can sometimes produce documentation that sounds polished but lacks project-specific accuracy. Without editing, this can make documentation look impressive but shallow or misleading.

### 7.4 Security and Privacy Concerns
If used carelessly, AI tools may encourage unsafe practices such as:
- exposing private data
- using insecure code patterns
- copying suggestions without verifying them

In this project, no sensitive personal data or private credentials were included in AI prompts.

### 7.5 Mismatch with Assignment Requirements
AI may generate content that does not precisely match course expectations, grading criteria, or instructor preferences. This is why manual alignment with the rubric is necessary.

## 8. Risk Mitigation Strategies
To reduce the risks above, the following steps were taken:

### 8.1 Manual Review
All AI-generated suggestions were reviewed before being accepted.

### 8.2 Testing
The logic, especially contact form validation and project interaction, was checked after updates.

### 8.3 Editing and Personalization
Generated documentation was edited and expanded so that it accurately reflected the actual project.

### 8.4 Assignment Alignment
Feedback from Assignment 2 was used as a guide to ensure the new improvements directly addressed past weaknesses.

### 8.5 Understanding Before Submission
The final code and documentation were reviewed to ensure the student could explain:
- what was changed
- why it was changed
- how the features work

## 9. Ethical and Academic Use of AI
AI was used in a supportive and transparent way. The intention was not to misrepresent authorship or avoid understanding, but to:
- improve quality
- strengthen communication
- reduce repeated mistakes
- learn from feedback more effectively

The final submission remains the student’s responsibility. Any AI-generated content included in the work was reviewed, adapted, and integrated intentionally.

## 10. Reflection on Assignment 2 Feedback
The Assignment 2 grading showed that the project was generally strong, but several areas needed improvement:
- email validation
- setup instructions
- technical detail
- user-friendly guidance
- depth of AI discussion

AI was especially helpful in responding to these exact weaknesses by:
- strengthening email validation logic
- expanding documentation structure
- adding more explanation and detail
- improving the discussion of AI risks and mitigation

This made AI useful not only for code generation, but also for reflective improvement based on grading feedback.

## 11. What Was Not Left Entirely to AI
The following parts still required personal judgment and were not simply accepted from AI without thought:
- project selection and descriptions
- visual structure and design direction
- whether features matched assignment goals
- what documentation needed to emphasize
- what errors from past grading needed direct attention

This distinction is important because AI can assist, but educational value depends on the student understanding and owning the final result.

## 12. Lessons Learned
Using AI in this project led to several important lessons:
1. AI is useful for refinement, but not enough by itself
2. Documentation quality can improve significantly with guided support
3. Validation logic should always be tested, not just generated
4. AI suggestions are most valuable when combined with grading feedback
5. Responsible AI use requires review, adaptation, and understanding

## 13. Conclusion
AI played a meaningful but controlled role in this project. It supported code review, documentation writing, debugging, and reflection. It helped improve the final quality of the submission, especially in areas that were weaker in Assignment 2.

However, AI was not treated as an automatic solution. Its outputs were reviewed, tested, and adapted before inclusion. This ensured that the final submission remained accurate, relevant, and aligned with academic expectations.

In conclusion, AI was used as a productivity and quality-enhancement tool, while the student remained responsible for understanding, reviewing, and finalizing all project work.
